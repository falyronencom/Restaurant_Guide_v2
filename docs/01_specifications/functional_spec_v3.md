## Часть I: Обзор проекта и бизнес-контекст

### 1.1 Проблема и решение

**Проблема:** В Беларуси отсутствует единая, современная платформа для
поиска и выбора заведений общепита, адаптированная под местную
специфику. Существующие решения либо устарели, либо не покрывают все
потребности пользователей и бизнеса.

**Решение:** Мобильное приложение для ресторанного бизнеса Беларуси, с фокусом на:

- Геолокационный поиск в шаговой доступности с честным ранжированием (без платных бустов позиций)
- Детальную информацию о заведениях с фотографиями и меню
- Прозрачную систему отзывов и рейтингов
- Инструменты привлечения клиентов для партнёров (акции, бронирование) — монетизация через ценность, а не через искажение выдачи
- Удобную модерируемую экосистему

### 1.2 Целевая аудитория

**Пользователи (User):**

- Возраст: 18-45 лет
- География: жители и гости 7 крупных городов Беларуси
- Потребности: быстрый поиск заведения поблизости, просмотр меню и цен, чтение отзывов

**Партнеры (Partner):**

- Владельцы и управляющие ресторанов, кафе, баров
- От небольших семейных кафе до сетевых ресторанов
- Потребности: привлечение клиентов, управление репутацией, аналитика

**Администраторы (Admin):**

- Внутренняя команда управления платформой
- На старте: основатель + возможно 1-2 модератора

### 1.3 Ключевые метрики успеха

**Технические:**

- Время загрузки главного экрана < 2 секунд
- Crash-free rate > 99%
- Размер приложения < 50MB

**Пользовательские:**

- 250-500 заведений к моменту публичного запуска (5-10% от общего числа)
- 1000+ активных пользователей к концу 2-го месяца
- Retention Day 7 > 20%

**Бизнес (Фаза 2 — после 6-12 мес. бесплатного периода):**

- 10+ платных подписок к концу 3-го месяца монетизации
- Конверсия из бесплатного в платный тариф > 30%

## Часть II: Техническая архитектура

### 2.1 Технологический стек

**Frontend (Mobile):**

- Flutter 3.x — кроссплатформенная разработка iOS/Android
- Provider (ChangeNotifier) для state management
- Dio для HTTP запросов
- Yandex MapKit для карт и геолокации
- Cached Network Image для оптимизации изображений

**Frontend (Admin Panel):**

- Flutter Web (desktop-first, PWA)
- Отдельный codebase: `admin-web/`
- Sidebar (363px) + content layout via AdminShell
- GoRouter с auth redirect guard
- Provider (ChangeNotifier) для state management

**Backend:**

- Node.js + Express
- PostgreSQL — основная база данных
- Redis — кэширование и rate limiting
- JWT — авторизация (access token 15min + refresh token 7d)
- Cloudinary — хранение и оптимизация фотографий заведений (3 разрешения: original/preview/thumbnail)
- Multer — загрузка аватаров пользователей (локальное хранение `backend/uploads/avatars/`)

**Инфраструктура:**

- SendGrid — email уведомления (запланировано)
- Docker — контейнеризация (запланировано)
- GitHub Actions — CI/CD (запланировано)
- Хостинг в регионе (Литва/Польша для низкой латентности)

## Часть III: Детальная функциональная спецификация

### 3.1 Пользовательские экраны (User Flow)

#### 3.1.1 Стартовый экран

**Назначение:** Точка входа, где пользователь формирует поисковый запрос.

**Элементы интерфейса:**

- **Кнопка выбора локации** — открывает экран с городами (Минск, Гродно, Брест, Гомель, Витебск, Могилёв, Бобруйск)
- **Поисковая строка** — для ввода названия конкретного заведения
- **Кнопка «Фильтр»** — открывает экран с детальными параметрами
- **Кнопка «Далее»** — переход к результатам с текущими параметрами

**Нижняя навигация (присутствует на всех экранах):**

- Поиск (текущий экран)
- Новости (Horizon 3-4 — когда появится контент-направление)
- Карта
- Избранное
- Профиль

**Логика работы:**

- При включенной геолокации автоматически определяется город, дефолтная сортировка — по расстоянию
- Кнопка «Далее» активна всегда, даже без выбора параметров
- При выключенной геолокации — предлагается выбор города, дефолтная сортировка — по рейтингу, расстояния на карточках не отображаются (см. Стратегия монетизации §2.7)
- Мягкий запрос геолокации: ненавязчивый баннер "Включите геолокацию, чтобы видеть заведения в шаговой доступности" — один раз за сессию

#### 3.1.2 Экран фильтров

**Структура (скроллируемый экран):**

**Расстояние от вас:**

- В радиусе 500 метров
- В радиусе 1 км
- В радиусе 3 км
- В радиусе 5 км
- В радиусе 10 км
- Весь город (по умолчанию)

*Примечание: При выключенной геолокации секция неактивна*

**Средний чек:**

- $ (бюджетно)
- $$ (средне)
- $$$ (дорого)
- $$$$ (премиум)

**Время работы:**

- До 22:00
- До утра
- 24 часа

**Категория заведения:**

- Кнопка «Все» (активна по умолчанию)
- Ресторан, Кафе, Кофейня, Фаст-фуд, Бар, Кондитерская, Пиццерия, Пекарня, Паб, Столовая, Кальянная, Боулинг, Караоке, Бильярд, Клуб (15 категорий)

**Категория кухни:**

- Кнопка «Все» (активна по умолчанию)
- Народная, Авторская, Азиатская, Американская, Вегетарианская, Японская, Грузинская, Итальянская, Смешанная, Европейская, Китайская, Восточная (12 кухонь)

**Дополнительные параметры:**

- Доставка еды
- Wi-Fi
- Организация банкетов
- Летняя терраса
- Залы для курящих
- Детская зона
- С домашними животными
- Есть парковка

**UI особенность:** Кнопка «Применить» зафиксирована внизу экрана при скролле

**Логика фильтрации:**

- Множественный выбор в категориях работает по принципу OR (любое совпадение)
- Между разными группами фильтров — AND (все условия должны выполняться)

#### 3.1.3 Экран списка заведений

**Структура экрана:**

**Верхняя панель (статичная):**

- Кнопка выбора локации
- Кнопка «Фильтр»
- Поисковая строка
- Кнопка «Назад»
- Кнопка «Сортировка» (По расстоянию, По рейтингу, По увеличению чека, По уменьшению чека). При отключённой геолокации «По расстоянию» недоступна
- Кнопка «Карта»

**Основная область (скроллируемая):**

- Счетчик результатов («Найдено 47 заведений»)
- Вертикальный список карточек заведений

**Карточка заведения содержит:**

- Главное изображение
- Название
- Рейтинг (звезды + число)
- Средний чек
- Категория заведения и тип кухни
- Время работы + индикатор «Открыто/Закрыто»
- Расстояние от пользователя (при включенной геолокации)
- Адрес
- Кнопка «В избранное»

**Ранжирование (см. Стратегия монетизации §2):**

- Честная выдача: никакие платные бусты не влияют на позицию в списке
- С геолокацией: дефолт — по расстоянию (PostGIS ST_Distance), тайбрейкеры: completeness → рейтинг → имя
- Без геолокации: дефолт — по рейтингу (двухэшелонная: порог 3 отзыва), тайбрейкеры: review_count → completeness → имя
- По цене: CASE $→1, $$→2, $$$→3, $$$$→4, NULL→5 (заведения без цены в конце)
- Карточки с активной акцией отображаются с бейджем акции (визуальное привлечение без изменения позиции)

#### 3.1.4 Детальная карточка заведения

**Структура (скроллируемый экран):**

**Галерея изображений (свайп/тап для полноэкранного просмотра):**

- Кнопки поверх изображения: «Назад», «В избранное», «Поделиться»

**Основная информация (на изображении):**

- Название
- Рейтинг и количество отзывов
- Средний чек
- Категория и кухня
- Часы работы с индикатором статуса
- Телефон (кликабельный)
- Email (кликабельный)

**Секция «Меню»:**

- Режим работы по дням
- Информация о специальных предложениях (завтраки, бизнес-ланчи)
- Галерея фотографий меню (горизонтальный скролл)

**Атрибуты (иконки с подписями, горизонтальный скролл):**

- Wi-Fi, Парковка, Терраса и другие удобства

**Карта:**

- Адрес (текстом)
- Расстояние от пользователя
- Интерактивная карта с маркером (тап открывает полноэкранную карту)

**Отзывы и рейтинги:**

- Общий рейтинг (большая цифра)
- Количество отзывов
- Карусель отзывов (горизонтальный скролл)
- Кнопка «Написать отзыв» (только для авторизованных)

### 3.3 Личный кабинет партнера

#### 3.3.1 Процесс регистрации партнера

**Шаг 1: Выбор категории заведения**

- Можно выбрать до 2 категорий
- Обязательное поле

**Шаг 2: Выбор типа кухни**

- Можно выбрать до 3 типов
- Обязательное поле

**Шаг 3: Основная информация**

- Название (обязательно)
- Описание (обязательно)
- Контактный телефон (обязательно)
- Ссылки на соцсети (опционально)
- Время работы будни/выходные (обязательно)
- Специальные часы (завтраки, обеды) — опционально
- Средний чек (опционально)
- Атрибуты заведения (чекбоксы)

**Шаг 4: Загрузка медиа**

- Фото интерьера/экстерьера: до 20 фото (обязательно минимум 1)
- Фото меню: до 20 фото (обязательно минимум 1)
- Выбор главного фото
- Установка порядка отображения

**Шаг 5: Адрес**

- Населенный пункт (выбор из списка городов)
- Улица
- Дом, корпус

**Шаг 6: Юридическая информация**

- Полное название юрлица (обязательно)
- УНП (с валидацией формата — 9 цифр)
- Контактное лицо и email (обязательно)
- Скан свидетельства о регистрации (опционально на этапе регистрации)

**Важно:** Система сохраняет прогресс как черновик. Отправка на модерацию — отдельное действие после заполнения всех шагов. Двухшаговое создание: POST → 'draft', POST /:id/submit → 'pending'.

**Верификация (см. Стратегия §5.3-5.4):** Claiming и верификация — независимые процессы. Для invite-link партнёров Legal Info запрашивается отложенно (баннер в кабинете). Верификация обязательна для платных функций. Полуавтоматическая проверка модератором: формат УНП + сверка с egr.gov.by.

#### 3.3.2 Личный кабинет партнера

**Главный экран:**

- Виньетки всех заведений партнера (горизонтальный скролл)
- Каждая виньетка показывает: статус модерации, главное фото, название, метрики

**Виньетка заведения содержит:**

- Статус (на модерации / активно / приостановлено)
- Превью изображения
- Основная информация
- Метрики: просмотры, избранное, отзывы
- Кнопка «Продвижение» (для активных)

**Действия с заведением:**

- Просмотр как видят пользователи
- Редактирование (без повторной модерации для большинства изменений)
- Добавить новое заведение

#### 3.3.3 Аналитика партнера

**Селектор периода:**

- Быстрый выбор: 7 дней, 30 дней, 90 дней
- Произвольный период

**Метрики с визуализацией:**

- Просмотры (view_count)
- Добавления в избранное (favorite_count)
- Отзывы и оценки (review_count, average_rating)
- Completeness score (прогресс-бар полноты карточки)
- Просмотры/клики акций (для платного тарифа)

**Особенности визуализации:**

- График показывает динамику выбранной метрики
- Подсветка периодов активной платной подписки
- Процентное сравнение с предыдущим периодом
- Автоматическая агрегация: по дням (до 30 дней), по неделям (31-90 дней), по месяцам (>90 дней)

#### 3.3.4 Управление отзывами

**Функциональность:**

- Просмотр всех отзывов с фильтрацией по периоду
- Возможность ответить на отзыв
- Индикация отзывов с ответами и без

### 3.4 Административная панель (Web)

#### 3.4.1 Технические требования

**Платформа:** Flutter Web (desktop-first, PWA)
**Доступ:** Только через защищенный веб-интерфейс
**Безопасность:** Email + пароль (2FA — планируется после MVP)
**Навигация:** Sidebar (363px) + content layout, GoRouter с auth guard

#### 3.4.2 Реализованная функциональность (MVP)

**Дашборд:**

- 4 метрических карточки: всего пользователей, новые за месяц, активные заведения, ожидают модерации
- 3 аналитических вкладки с визуализацией (графики):
  - **Пользователи:** тренд регистраций по времени, распределение по ролям
  - **Заведения:** динамика создания, распределение по статусам и городам
  - **Отзывы:** тренд публикаций, распределение по рейтингам
- Селектор периода: 7 дней / 30 дней / 90 дней / произвольный
- Процентное сравнение с предыдущим периодом
- Автоматическая агрегация: по дням (до 30 дней), по неделям (31-90), по месяцам (>90)

**Модерация заведений:**

- Вкладка «Ожидающие» — очередь заявок, отсортированная по времени (старые первые), с пагинацией
- Вкладка «Активные» — список активных заведений с поиском и сортировкой
- Вкладка «Отклонённые» — история отклонений с причинами
- Полноэкранный просмотр заявки: все данные, фотогалерея, юридическая информация
- Действия: Одобрить / Отклонить (с обязательным полем причины)
- Заблокировать активное заведение (с причиной) / Снять блокировку
- Поиск по всем статусам одновременно

**Управление отзывами:**

- Список всех отзывов (включая скрытые и удалённые)
- Фильтры: статус (видимые / скрытые / удалённые), рейтинг, период, текстовый поиск
- Действия: скрыть / показать отзыв, мягкое удаление
- Пагинация

**Журнал действий:**

- Хронологический лог всех административных действий
- Фильтры: тип действия, тип объекта, пользователь, период
- Поля записи: кто совершил, когда, что сделал, с каким объектом
- Пагинация

#### 3.4.3 Планируется после MVP

- Двухфакторная аутентификация (2FA) при входе
- Множественные уровни доступа (модератор, старший модератор, супер-админ)
- Экспорт данных из аналитики
- Управление подписками и платежами партнёров (таблица plans, не enum)
- Система уведомлений администраторам
- Admin-эндпоинты для claiming: смена partner_id, upgrade-to-partner, invite-link генерация
- Полуавтоматическая верификация партнёров (форматные проверки + ссылка на egr.gov.by)
- AI-модерация отзывов

## Часть IV: Система монетизации

> **Основной документ:** `docs/strategy/monetization_strategy_v0.4.md`
> Эта секция — краткое изложение. При расхождениях — приоритет у стратегии.

### 4.1 Философия

Честная выдача + монетизация через инструменты. Nirivio не продаёт позиции в выдаче. Никаких процентных бустов к ранжированию. Все сортировки (расстояние, рейтинг, цена) отражают реальные данные. Монетизация строится на инструментах привлечения клиентов, которые создают ценность и для партнёра, и для пользователя. На карточке заведения никогда не рекламируются конкуренты.

### 4.2 Два уровня

**Бесплатный — «Присутствие»:**

- Полная карточка без ограничений (фото, описание, меню, часы работы)
- Полная аналитика (просмотры, избранное, клики)
- Ответы на отзывы без ограничений
- Честная позиция в выдаче
- Лимиты фото: пересмотреть в сторону увеличения (текущие 10+10 — наследие старой модели)

**Платный — «Продвижение»:**

Предварительно один тариф (финальное решение привязано к ценовой точке — отдельная сессия):

- **Акции** — промо-предложения, видимые как бейдж на карточке + бейдж на пине карты. Не влияют на позицию в выдаче. Отдельная метрика кликов.
- **Бронирование** — дата, время, гости → партнёр подтверждает/отклоняет. Без комиссий. Релевантно для Dining и Entertainment, не для Quick-сегмента.
- *(Будущее)* Push-уведомления пользователям из «Избранного»

**Подписка на уровне establishment**, не user. Это позволяет: разным заведениям одного партнёра иметь разные тарифы, сетевую подписку как batch-UPDATE.

### 4.3 Три фазы запуска

**Фаза 1 (запуск → 6-12 мес.):** Все функции бесплатны (включая акции и бронирование). Наполнение базы, привлечение партнёров.

**Фаза 2 (6-12 мес. → 1.5-2 года):** Один платный тариф. Акции и бронирование переходят в подписку. Бесплатный уровень остаётся щедрым.

**Фаза 3 (когда есть данные):** Дифференциация тарифов на основе данных Фазы 2. Раздел акций в "Новостях", расширенная аналитика.

### 4.4 Claiming: наполнение базы

Seed-карточки (создаются Координатором по публичным данным) + claiming workflow:

- **Фаза 1:** Ручной claiming через админку (10-15 заведений)
- **Фаза 1.5:** Invite-link claiming (когда seed > 50)
- **Фаза 3:** Full self-service с верификацией (Horizon 3-4)

Claiming и верификация — независимые процессы. Партнёр получает карточку сразу, документы (УНП, юрлицо) — отложенно, обязательны для платных функций.

### 4.5 Completeness Score

Числовой показатель полноты карточки (`base_score`). Используется как тайбрейкер (при равном расстоянии — более полная карточка выше) и как мотиватор (прогресс-бар в кабинете). Считается по опциональным полям: описание, price_range, атрибуты, телефон, email, website. Seed-карточки имеют естественно низкий score.
